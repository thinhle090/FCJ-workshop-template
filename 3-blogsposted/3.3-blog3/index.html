[SECURITY/Architecture] SEPARATING TRANSACTIONAL AND ANALYTICAL WORKLOADS FOR REAL-TIME ANALYTICS: LESSONS FROM OLDCASTLE WITH AMAZON AURORA AND AMAZON QUICKSIGHT

Introduction

While researching real-time analytics architectures on Amazon Web Services (AWS), I came across an AWS Architecture Blog article describing how Oldcastle APG designed its analytics platform on AWS. The article focuses on separating transactional (OLTP) workloads from analytical workloads, enabling real-time data analysis without impacting the performance of the core ERP system.

This article summarizes and analyzes the architecture proposed by AWS, highlighting the role of each component in building a real-time analytics platform based on a Scalable Architecture approach that provides high availability, scalability, and efficient data processing.

Background

Manufacturing and logistics enterprises continuously generate transactional data, including:

  • Sales orders
  • Inventory transactions
  • Production management
  • Customer management

Business departments require near real-time visibility into operational data to support decision-making.

However, when Business Intelligence (BI) tools query the transactional database directly, the ERP system may experience several issues, including:

  • Performance degradation
  • Increased processing workload
  • Longer response times
  • Negative impacts on business operations

To address these challenges, AWS proposes an architecture that completely separates transactional processing from analytics through a Real-time Data Streaming approach.

1. Real-Time Data Ingestion

Challenge

Changes generated by the ERP system must be synchronized continuously with the analytics platform without affecting the performance of the transactional database.

Analysis

Oldcastle uses Infor Data Fabric Stream Pipelines to capture events generated by the ERP system.

Operations such as:

  • Insert
  • Update
  • Delete

are streamed immediately into the AWS environment instead of waiting for traditional batch processing.

Benefits

  • Near real-time data synchronization.
  • Reduced data latency.
  • Eliminates direct analytical queries against the ERP database.

2. Secure Connectivity to the AWS Environment

Challenge

Infor Cloud ERP cannot directly access the Amazon Aurora database deployed inside a private subnet.

Allowing direct access would increase the security risk of the network infrastructure.

Analysis

AWS implements a secure connectivity architecture consisting of:

  • Amazon Route 53
  • Network Load Balancer (NLB)
  • EC2 Router (RDS Proxy Router)
  • Amazon RDS Proxy

The Network Load Balancer uses static Elastic IP addresses to receive traffic from the ERP system.

EC2 Router instances perform Network Address Translation (NAT) and forward incoming traffic to Amazon RDS Proxy, which then manages all database connections to Amazon Aurora PostgreSQL.

Benefits

  • Protects the database within a private subnet.
  • Reduces the number of direct database connections.
  • Supports connection pooling.
  • Improves scalability and failover capabilities.

Figure 1. Oldcastle Real-Time Analytics Architecture on AWS

Figure 1 illustrates the real-time analytics architecture deployed between Infor Cloud ERP and Amazon Web Services.

The workflow consists of the following steps:

  1. Infor ERP generates transactional events and streams data through Infor Data Fabric Stream Pipelines.
  2. Traffic is routed through Amazon Route 53 and the Network Load Balancer into the AWS environment.
  3. EC2 Router instances receive the incoming traffic and forward it through Amazon RDS Proxy.
  4. Amazon RDS Proxy manages database connections to Amazon Aurora PostgreSQL, improving performance and scalability.
  5. Amazon Aurora stores transactional data and supports flexible querying using the JSONB data type.
  6. Amazon QuickSight connects to Aurora to build real-time analytical dashboards.
  7. AWS Lambda works together with Amazon API Gateway to authenticate users and generate embedded dashboard URLs.
  8. Dashboards are embedded directly into the Infor ERP interface through Embedded Analytics, allowing users to access reports without leaving the ERP application.

This architecture completely separates transactional workloads from analytical workloads while ensuring that analytical data remains continuously synchronized without impacting ERP performance.

3. Data Storage and Processing

Challenge

Real-time streaming data often has flexible and evolving structures.

Traditional relational storage models may reduce flexibility when processing analytical workloads.

Analysis

Oldcastle uses Amazon Aurora PostgreSQL with the JSONB data type to store streaming data.

PostgreSQL’s native JSON functions enable:

  • Flexible querying.
  • Semi-structured data analysis.
  • Easy schema evolution.

Additionally, Aurora is deployed using a Multi-AZ architecture to provide high availability.

Benefits

  • Flexible data storage.
  • Efficient query performance.
  • Improved scalability.
  • High availability.

4. Data Visualization with Amazon QuickSight

Analysis

Amazon QuickSight is used to build real-time analytical dashboards.

To improve performance, the solution leverages:

  • SPICE In-memory Engine
  • Embedded Dashboards
  • Amazon API Gateway
  • AWS Lambda

Through Embedded Analytics, dashboards are integrated directly into the Infor ERP interface.

Users can access analytical reports within their familiar working environment without switching to a separate BI platform.

Benefits

  • Faster dashboard performance.
  • Improved user experience.
  • Fine-grained access control.
  • Seamless integration with ERP systems.

Conclusion

The architecture presented by AWS demonstrates that separating transactional workloads from analytical workloads is an effective strategy for organizations requiring real-time analytics.

By combining Infor Data Fabric Stream Pipelines, Amazon Aurora PostgreSQL, Amazon RDS Proxy, and Amazon QuickSight, organizations can minimize the workload on ERP systems while delivering scalable, high-performance, near real-time analytics.

This architecture serves as a practical reference for modern ERP environments, particularly in manufacturing, logistics, and supply chain industries, where continuously changing data requires timely analysis and informed decision-making.
