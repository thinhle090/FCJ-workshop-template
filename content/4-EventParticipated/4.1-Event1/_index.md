#### 2.1 Presentation: Automated Prompt Engineering

**Speaker: Nguyen Tuan Thinh**

The presentation explained the importance of Prompt Engineering in improving the quality of responses generated by Large Language Models (LLMs). It also introduced practical principles and techniques for creating more effective prompts.

**Why Prompt Engineering Matters:**

- Vague prompts often produce inaccurate or incomplete responses.
- Poorly designed prompts consume unnecessary tokens.
- Ambiguous instructions reduce both response quality and productivity.

**Elements of an Effective Prompt:**

| Component | Description |
|---|---|
| Role | Defines the AI's persona or responsibility |
| Instruction | Specifies the task the AI should perform |
| Context | Provides background information |
| Input Data | Data that the AI needs to process |
| Output Format | Defines how the response should be presented |
| Example | Demonstrates expected input and output |
| Constraints | Specifies rules and limitations |

**Guidelines for Writing Better Prompts:**

- Write clear and specific instructions.
- Use direct and concise language.
- Describe the required task accurately.
- Allow the AI to respond with "I don't know" when necessary.
- Break complex tasks into smaller steps.

**Token Economy:**

- Tokens are the basic processing units used by LLMs.
- Costs are calculated based on input and output tokens.
- Vietnamese generally requires more tokens than English.

**Advanced Prompting Techniques:**

- **Chain-of-Thought (CoT):** Guides the AI to reason step by step.
- **Tree-of-Thoughts (ToT):** Explores multiple reasoning paths.
- **Self-Consistency:** Compares different reasoning results to select the best answer.
- **Retrieval-Augmented Generation (RAG):** Enhances responses using external knowledge sources.
- **Role Prompting:** Assigns a specific role to improve contextual responses.

**Proptimizer Tool:**

Proptimizer is a browser extension that automatically optimizes prompts using an AWS Serverless architecture:

#### 2.2 Mind Map: Effective AI Interaction & AWS Application Architecture

**Improving AI Interaction**

The mind map introduced the **KFC (Knowledge – Format – Constraints)** framework, which helps users create better prompts by providing sufficient context, defining the desired output format, and specifying clear constraints.

**Recommended Practices:**

- Divide complex problems into smaller tasks.
- Ask AI to suggest multiple solutions.
- Compare different responses before making a decision.
- Provide structured and well-organized input data.

**Advanced AI Techniques:**

- Chain-of-Thought (CoT)
- Tree-of-Thoughts (ToT)
- Self-Consistency
- Retrieval-Augmented Generation (RAG)
- Role Prompting

#### 2.3 BMAD Methodology (Build More Architect Dreams)

BMAD is an open-source AI development framework that provides a structured workflow for designing and building AI-driven applications. It helps development teams organize their work more efficiently throughout the software development lifecycle.

**Key Features:**

- A structured workflow covering Analysis → Planning → Architecture → Implementation.
- More than 12 specialized AI agents supporting different development stages.
- Built-in Agile development practices.
- AI assistant available through the **bmad-help** command.
- **Party Mode** for collaboration between multiple AI agents.

**Core Modules:**

| Module | Description |
|---|---|
| BMM | Core framework containing 34 development workflows |
| BMB | Build custom AI agents |
| TEA | Testing and automation support |
| BMGD | Game development module |
| CIS | Innovation and Design Thinking support |

**Quick Installation** *(Requires Node.js 20+, Python 3.10+, and uv)*

```bash
npx bmad-method install
```
