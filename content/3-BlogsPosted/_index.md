### [BLOG 1 - Don't Let the Cloud Become a Blind Spot: Designing a Ransomware-Resilient AWS Infrastructure](3.1-Blog1/)

This blog introduces the core principles of designing secure architectures on Amazon Web Services (AWS) based on the **Security by Design** approach. It focuses on reducing the attack surface and strengthening defense mechanisms against ransomware attacks. The article analyzes key security practices, including temporary privilege management, network traffic isolation, integrating security into the software development lifecycle, and centralized security monitoring. Through these concepts, it highlights how architectural decisions can significantly improve an organization's ability to detect, prevent, and respond to cybersecurity threats in cloud environments.

### [BLOG 2 - Secure Access Control for Multi-user RAG Applications with Amazon Bedrock and Verified Permissions](3.2-Blog2/)

This blog introduces a secure access control architecture for multi-user Retrieval-Augmented Generation (RAG) applications on Amazon Web Services (AWS). It focuses on how Amazon Cognito, Amazon API Gateway, AWS Lambda, Amazon Verified Permissions, and Amazon Bedrock can be combined to implement multi-layer authentication and authorization following the **Defense in Depth** principle. The article also explains the importance of centralized policy management in improving security, scalability, and preventing unauthorized data access across different user groups in Generative AI applications.

### [BLOG 3 - Separating Transactional and Analytical Workloads for Real-Time Analytics: Lessons from Oldcastle with Amazon Aurora and Amazon QuickSight](3.3-Blog3/)

This blog presents the real-time analytics architecture developed by AWS for Oldcastle APG, which separates transactional (OLTP) workloads from analytical workloads. The article focuses on how services such as Amazon Aurora PostgreSQL, Amazon RDS Proxy, Amazon QuickSight, and Infor Data Fabric Stream Pipelines are integrated to build a scalable and high-performance analytics platform. It also demonstrates how isolating the analytics layer from the ERP system reduces the workload on transactional databases while enabling near real-time business intelligence and decision-making.
